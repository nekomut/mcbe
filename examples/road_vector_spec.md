# 道路ベクトルタイル仕様メモ

地理院ベクトルタイル `experimental_bvmap` の道路関連情報。
gen_terrain.py での道路・橋判定の設計資料。

## タイル URL

```
https://cyberjapandata.gsi.go.jp/xyz/experimental_bvmap/{z}/{x}/{y}.pbf
```

### 状態コード（4桁目）の意味

| 4桁目 | 状態     | 備考                           |
|-------|----------|--------------------------------|
| 1     | 通常部   | 地上の道路                     |
| 2     | 雪覆い   | スノーシェッド等               |
| 3     | 橋・高架 | **橋の判定に使える可能性あり** |
| 4     | トンネル | 地下。MC では非表示が妥当      |

## road レイヤーのプロパティ

z=14–16 で取得できる属性:

| プロパティ | 型  | 説明         | 値の例                                                    |
|------------|-----|--------------|-----------------------------------------------------------|
| ftCode     | int | 地物コード   | 上記参照                                                  |
| rdCtg      | int | 道路分類     | 0=国道, 1=都道府県道, 2=市区町村道, 3=高速道路, 5=その他  |
| rnkWidth   | int | 幅員区分     | 0=3m未満, 1=3–5.5m, 2=5.5–13m, 3=13–19.5m, 4=19.5m以上 |
| Width      | int | 実幅員(m)    | 数値                                                      |
| motorway   | int | 高速道路区分 | 0=一般, 1=高速                                            |
| tollSect   | int | 有料区間     | 0=無料, 1=有料                                            |
| lvOrder    | int | 階層順       | 立体交差の描画順序                                        |
| medSect    | int | 分離帯       | 0=なし, 1–98=幅(m)                                       |

## 現在の処理フロー

```
Pass 1: z=15 道路縁線 (27xx) → label_map にラスタライズ
         ↓
z=15 距離変換 (z15_dist)
         ↓
Pass 2: z=16 線 (22xx + 27xx) → z=15 近傍のみ label_map に追加
         同時に 22xx を center_mask に記録
         ↓
全エッジ距離変換 → Voronoi 境界 → 道路面充填
         ↓
交差点充填 (3+ ラベル集合)
         ↓
橋判定: road_filled & water_mask
```

## --debug レッドストーン配置

gen_terrain.py に `--debug` を指定すると `centerlinemap` が terrain.json に出力される。
place_block.py はこれを読み込み、橋フェーズの後に独立した「centerline」フェーズで
レッドストーンブロックを配置する。

### データの流れ

```
gen_terrain.py --debug
  → z=16 中心線 (ftCode 2201/2221) を center_mask にラスタライズ
  → center_mask & (道路面 | 橋面) → centerlinemap として JSON 出力

place_block.py
  → terrain → building → bridge → centerline の順にフェーズ実行
  → centerline フェーズ: 橋面があれば橋の高さ、なければ地形の高さに
    レッドストーンブロックを /setblock で配置
```

### centerlinemap の仕様

- 値: 0 = なし, 1 = 中心線あり
- ソース: z=16 ベクトルタイルの ftCode 2201/2221（道路中心線）
- フィルタ: z=15 縁線から `max_road_half_width` 以内の線のみ採用
- マスク: 道路面 (`surface == SURFACE_ROAD`) または橋面 (`bridge_mask`) 上のみ

### place_block.py での配置ロジック

- 橋面がある (`bridgemap[z][x] > 0`): 橋面高さの最上面にレッドストーン
- それ以外: 地形高さ (`heightmap`) の最上面にレッドストーン
- 専用の `centerline.progress` でレジューム対応

## 改善の検討ポイント

### 橋の判定
- 現在: `road_filled & water_mask` で検出（水域上の道路 = 橋）
- 課題: 水域がない橋（高架・陸橋）を検出できない
- 改善案: ftCode の4桁目 = 3（橋・高架）を使って直接判定
  - 2703, 2713, 2723, 2733 = 橋・高架の縁線
  - これらの縁線で囲まれた道路面 = 橋

### トンネルの判定
- ftCode の4桁目 = 4（トンネル）
- 2704, 2714, 2724, 2734 = トンネルの縁線
- MC ではブロックを配置しない（地下通過）のが妥当

### 道路幅の利用
- `rnkWidth` / `Width` プロパティで実際の道路幅が分かる
- Voronoi の max_road_half_width を道路ごとに設定できる可能性

### 未使用の ftCode
- 2702, 2712, 2722, 2732（雪覆い）— 通常部と同じ扱いで良い
- 2713, 2733（庭園路・石段の橋）— 現在は Voronoi エッジに含めていない
